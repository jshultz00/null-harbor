Container Migration Summary

Migrating from VMs to containers drops estimated RAM from 171 GB to ~64 GB (full) or ~20 GB (core profile). Linux services shed 80–95% of their VM RAM footprint. Windows services via dockur/windows still need real RAM but save Proxmox overhead.

Architecture Changes

pfSense VMs → nftables/Debian containers. Both firewalls replaced with ~256 MB containers. Lose the web UI but gain portability and near-zero RAM cost.
VyOS VM → FRRouting container. Full BGP/OSPF support at 256 MB vs 1 GB VM.
Splunk 16 GB VM → 4 GB Docker container (official splunk/splunk image).
Security Onion 32 GB VM → 8 GB Docker mode. SO ships its own container stack — use standalone install mode.
Linux web/mail/rsyslog/workstation containers each drop to 256–512 MB from 2–4 GB VMs.
Windows targets → dockur/windows (ghcr.io/dockur/windows). KVM-in-Docker — requires a Linux host with /dev/kvm. RAM savings modest but portability is the win.
Win7 (WKS-01) may need a manually supplied ISO for dockur/windows. Test before relying on it for EternalBlue scenarios.
