# .env.example — Environment Variables Reference

This document describes all variables in `.env.example`. Copy to `.env` and fill in values before running `make build`.

Variables marked **REQUIRED** must be changed from defaults. Variables marked **OPTIONAL** have safe defaults.

---

## Domain & Credentials

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `RANGE_PASSWORD` | `P@55w0rd!` | Optional | Default password for all range accounts (Windows local, AD users, Linux users) |
| `AD_DOMAIN` | `secure.net` | Optional | Active Directory domain name (FQDN) |
| `AD_NETBIOS` | `SECURE` | Optional | NetBIOS domain name (15 char max) |

> Changing `RANGE_PASSWORD` updates Windows Administrator and domain user passwords. Linux container passwords are baked into Dockerfiles and must be updated there separately.

---

## Windows / MSSQL

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `ACCEPT_EULA` | `Y` | Optional | MSSQL EULA acceptance — must be `Y` |
| `MSSQL_SA_PASSWORD` | — | **REQUIRED** | MSSQL `sa` account password. Must meet SQL Server complexity requirements (min 8 chars, uppercase, lowercase, digit, special) |

---

## Wazuh

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `WAZUH_VERSION` | `4.9.2` | Optional | Wazuh stack version (manager, indexer, dashboard images must all match) |
| `WAZUH_ENROLLMENT_PSK` | — | **REQUIRED** | Pre-shared key for Wazuh agent auto-enrollment (`authd.pass`). Arbitrary string, 32+ chars recommended |
| `WAZUH_API_PASSWORD` | — | **REQUIRED** | Password for the `wazuh-wui` API user. Used by the dashboard to query the manager API |
| `WAZUH_INDEXER_PASSWORD` | — | **REQUIRED** | OpenSearch initial admin password. Must meet complexity requirements |

---

## Infrastructure

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `SETUP_SERVER` | `10.0.0.1` | Optional | IP of the scenario container. Windows VMs download `setup.ps1` from `http://${SETUP_SERVER}:8000/windows/<machine>/` on first boot |
| `WAZUH_MANAGER` | `10.0.0.5` | Optional | IP of the Wazuh manager. Injected into Linux Dockerfiles and Windows `setup.ps1` scripts for agent registration |
| `COMMANDLY_SERVER` | `http://10.0.0.1:8080` | Optional | Full URL of the Saffron server. Injected into scenario `env_vars.sh` and all container environments |

> Do not change `SETUP_SERVER`, `WAZUH_MANAGER`, or `COMMANDLY_SERVER` unless you have changed the corresponding static IPs in `docker-compose.yml`.

---

## WireGuard

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `WIREGUARD_SUBNET` | `10.99.0.0/24` | Optional | Subnet assigned to WireGuard peers. Must not overlap with any range segment |

---

## Example `.env.example` File

```dotenv
# Domain
RANGE_PASSWORD=P@55w0rd!
AD_DOMAIN=secure.net
AD_NETBIOS=SECURE

# Windows / MSSQL
ACCEPT_EULA=Y
MSSQL_SA_PASSWORD=

# Wazuh
WAZUH_VERSION=4.9.2
WAZUH_ENROLLMENT_PSK=
WAZUH_API_PASSWORD=
WAZUH_INDEXER_PASSWORD=

# Infrastructure (do not change unless IPs changed in docker-compose.yml)
SETUP_SERVER=10.0.0.1
WAZUH_MANAGER=10.0.0.5
COMMANDLY_SERVER=http://10.0.0.1:8080

# WireGuard
WIREGUARD_SUBNET=10.99.0.0/24
```
