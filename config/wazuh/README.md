# config/wazuh/ — Wazuh SIEM Stack Configuration

This directory contains configuration files and TLS certificates for the three Wazuh containers: manager, indexer (OpenSearch), and dashboard.

---

## Files

| File | Container | Purpose |
|------|-----------|---------|
| `ossec.conf` | wazuh.manager | Main Wazuh manager configuration |
| `authd.pass` | wazuh.manager | Pre-shared key for agentless enrollment |
| `wazuh.indexer.yml` | wazuh.indexer | OpenSearch node configuration |
| `opensearch_dashboards.yml` | wazuh.dashboard | Dashboard connection to indexer |
| `certs/` | all three | TLS certificate chain (generated by `make certs`) |

---

## TLS Certificate Chain

All communication between Wazuh components is TLS-encrypted. Certificates are generated by `make certs` using OpenSearch's `wazuh-certs-tool.sh` or plain OpenSSL. The chain:

```
root-ca.pem                (self-signed CA, 4096-bit RSA, 10yr expiry)
├── wazuh.indexer.pem      (SAN: wazuh.indexer, 10.0.0.6, 10.50.50.6)
├── wazuh.manager.pem      (SAN: wazuh.manager, 10.0.0.5, 10.50.50.5)
├── wazuh.dashboard.pem    (SAN: wazuh.dashboard, 10.0.0.7, 10.50.50.7)
├── filebeat.pem           (client cert — used by Filebeat on manager to auth to indexer)
└── admin.pem              (admin client cert — used by securityadmin.sh bootstrap)
```

All `.pem` files are bind-mounted into their respective containers as `/etc/ssl/<name>.pem`.

`make certs` writes a `.stamp` file after generation to prevent accidental regeneration (which would break all agent connections). To force regeneration: `rm config/wazuh/certs/.stamp && make certs`.

## ossec.conf — Key Sections

### Global Settings

```xml
<global>
  <jsonout_output>yes</jsonout_output>
  <alerts_log>yes</alerts_log>
  <logall>yes</logall>
  <email_notification>no</email_notification>
</global>
```

`logall=yes` captures all events (not just alerts) to `/var/ossec/logs/archives/` — essential for training so defenders can see the full event stream.

### Cluster (single-node)

```xml
<cluster>
  <name>range-cluster</name>
  <node_name>master-node</node_name>
  <node_type>master</node_type>
  <key><!-- 32-char random key --></key>
  <port>1516</port>
  <bind_addr>0.0.0.0</bind_addr>
  <nodes>
    <node>wazuh.manager</node>
  </nodes>
  <hidden>no</hidden>
  <disabled>no</disabled>
</cluster>
```

### Agent Enrollment (authd)

```xml
<auth>
  <disabled>no</disabled>
  <port>1515</port>
  <use_source_ip>no</use_source_ip>
  <use_password>yes</use_password>   <!-- Requires authd.pass PSK -->
  <purge>yes</purge>
  <ssl_verify_host>no</ssl_verify_host>
  <ssl_auto_negotiate>no</ssl_auto_negotiate>
</auth>
```

### Syslog Listener (for firewall logs)

```xml
<remote>
  <connection>syslog</connection>
  <port>514</port>
  <protocol>udp</protocol>
  <allowed-ips>10.50.50.0/24</allowed-ips>
</remote>
```

This listener receives syslog only from the SIEM segment (`10.50.50.8` is rsyslog). The management network (`10.0.0.0/24`) is not an allowed source — rsyslog forwards via its SIEM-segment interface, and the rsyslog config drops any management-sourced logs before forwarding as defense-in-depth.

> **Invariant:** Wazuh must never receive or index events with 10.0.0.x as source or destination. Enforcement layers: (1) nftables accepts management traffic before log rules, (2) rsyslog drops any 10.0.0.x logs before forwarding, (3) this listener does not accept connections from 10.0.0.0/24.

  <feed-update-interval>60m</feed-update-interval>
</vulnerability-detection>

## authd.pass

Contains only the pre-shared key (one line, no trailing newline):

```
<value of WAZUH_ENROLLMENT_PSK from .env>
```

Agents use this PSK when registering with `wazuh-authd` on port 1515. It prevents unauthorized agents from enrolling. The PSK is injected into Linux Dockerfiles and Windows `setup.ps1` scripts at build time via the `WAZUH_ENROLLMENT_PSK` env var.

## Agent Enrollment in Linux Dockerfiles

Linux containers enroll with Wazuh at container start via `wazuh-agentd`:

```bash
/var/ossec/bin/agent-auth \
  -m 10.50.50.5 \
  -P "${WAZUH_ENROLLMENT_PSK}" \
  -A "$(hostname)"
```

This runs in the container's entrypoint before the primary service starts. Agent registration is idempotent — if the agent name is already registered, authd returns the existing agent ID.
